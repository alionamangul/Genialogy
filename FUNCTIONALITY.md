# Функциональное описание сайта

> Что умеет сайт с точки зрения пользователя. Каждая вкладка — отдельный раздел.
> Последнее обновление: 2026-06-20

Сайт — одностраничное приложение с верхним меню-вкладками. Все экраны работают без перезагрузки.

## Навигация (вкладки)

`Дерево · Список · Медиа · Документы · История · Факты · География · Книга`

---

## 1. Дерево

Главный экран — интерактивное генеалогическое дерево.

- **Сфокусированный вид:** по умолчанию показывает не все 295 человек, а только релевантных
  предков и потомков выбранной пары (по умолчанию — Армен Шахназарян + Алёна Мангул).
- **Узлы-аватарки:** каждый человек — круглый узел с фото (или инициалами), именем и годами жизни.
- **Обе ветви равноправны:** предки и со стороны Шахназарян, и со стороны Мангул/Курудимов
  показаны как основные (золотое кольцо), а не приглушённо.
- **Навигация по полотну:** перетаскивание (pan), масштаб колесом/жестами и кнопками `+` / `−`.
- **Кнопка «Всё дерево»** — переключение между сфокусированным и полным видом.
- **Клик по человеку** открывает боковую панель-карточку.
- **Линии связи** рисуются между родителями, парами и детьми.

## 2. Список

Алфавитный список всех людей с поиском.

- **Поиск по имени** в шапке (фильтрует в реальном времени).
- Клик по строке открывает карточку человека.

## 3. Медиа

Галерея фотографий, сгруппированная по фамилиям.

- Карточки людей с обложкой-фото и счётчиком («3 фото»).
- Клик разворачивает все фото человека.
- **Загрузка фото** кнопкой «＋»: можно выбрать несколько файлов с устройства (телефон/компьютер).
  Фото сжимается до 1600px перед отправкой (экономия трафика), сохраняется и сразу появляется в галерее.
- Клик по фото открывает его в полноэкранном просмотре (лайтбокс).

## 4. Документы

Галерея отсканированных документов (свидетельства, справки, письма, PDF).

- Группировка по людям, обложка из первого документа.
- Внутри карточки — вкладки по типам документов.
- **Загрузка документов** через модальное окно: выбор человека, тип, заголовок, один или несколько файлов.
- Поддерживаются изображения и PDF.

## 5. История

Текстово-фото блоки с историческими справками о семьях и ключевых событиях рода.

## 6. Факты

Отдельные карточки-факты с фотографиями — интересные эпизоды и сведения о членах семьи.

## 7. География

Визуализация мест, связанных с родом (места рождения, проживания).

## 8. Книга

Оформленная «семейная летопись» в виде книги с перелистыванием страниц.

- Две книги: **Шахназарян** (коричневая) и **Курудимовы / Мангул** (синяя).
- **На компьютере:** разворот из двух страниц с 3D-анимацией перелистывания.
- **На телефоне:** одна страница во весь экран, плавная смена с fade-переходом,
  счётчик «стр. X / N». Это специально сделано, потому что разворот на узком экране нечитаем.
- Содержит: титульный лист, текстовое древо рода (├─ └─), исторические справки, фотографии.
- Навигация стрелками `‹` `›`, на компьютере — клавиши ←→ и Escape.
- **Скачать PDF** — печать книги через системный диалог (`window.print()`).

## Карточка человека (боковая панель)

Открывается кликом по человеку в дереве, списке или галерее.

- Аватар + галерея фотографий с кнопкой загрузки.
- Редактируемые поля: имя, фамилия, второе имя, пол, даты и места рождения/смерти,
  род занятий, место жительства, заметки.
- Списки: родители, супруги, дети, братья/сёстры — с переходом по клику.
- Документы человека (превью + клик на полный просмотр).

### Как работает сохранение

- Правка поля сохраняется автоматически: при вводе с задержкой 800 мс (debounce),
  плюс принудительное сохранение при закрытии панели.
- Это сделано так, потому что на мобильных событие `onblur` срабатывает ненадёжно.
- Локально данные пишутся через `POST /api/update-data` в `data.json`.
- На проде тот же запрос уходит в serverless-функцию, которая коммитит `data.json` в GitHub.

## Что НЕ работает / ограничения

- На проде редактирование требует настроенных `GITHUB_TOKEN` и `BLOB_READ_WRITE_TOKEN` в Vercel.
- Нет аутентификации — редактировать может любой, у кого есть ссылка (см. [BACKLOG.md](BACKLOG.md)).
- `serve.py` (локальный сервер) на Vercel не запускается — там работают только `api/*.py`.
