### 🗑️ Удаление фото из карточки — чек-лист

**Цель:** дать возможность удалить фотографию из карточки человека (и из галереи «Медиа»).

**Сделать:**
- [ ] UI: кнопка «удалить» (✕/корзина) на каждой миниатюре фото — в боковой карточке человека и в развёрнутой карточке вкладки «Медиа». На мобильном — достаточно крупная зона нажатия.
- [ ] Подтверждение перед удалением (фото восстановить нельзя).
- [ ] Фронтенд: убрать ссылку из `p.photoIds[]`, перерисовать карточку/галерею, сохранить данные.
- [ ] Бэкенд-функция `POST /api/delete-photo` (Node, `api/delete-photo.js`): принять `personId` + ссылку на фото; убрать из `data.json` в Blob; удалить сам объект из Blob через `del(url)` из `@vercel/blob` (только если ссылка — Blob URL).
- [ ] Локальный аналог в `serve.py` (удаление файла из `/tmp/family-tree/media` + правка data.json).

**Проверить (важные нюансы):**
- [ ] Ссылки в `photoIds[]` бывают трёх видов: голый числовой id (старое `./media/<id>.jpg`), имя файла, полный Blob URL. Удаление должно корректно работать со всеми; **физически удалять только Blob-объекты** (старые фото из репозитория `del()` не трогает — их нет в Blob).
- [ ] Главное фото = `photoIds[0]`. После удаления первого элемента обложка в дереве/списке/медиа должна корректно перерисоваться (или показать инициалы, если фото не осталось).
- [ ] Одно и то же фото может встречаться у нескольких людей? Если да — не удалять Blob-объект, пока на него ссылается кто-то ещё (или принять, что фото на человека уникально — проверить текущее поведение загрузки).
- [ ] Гонка с автосохранением карточки (`update-data` шлёт весь `data.json`) — порядок операций, чтобы удаление не затёрлось параллельным сохранением.
- [ ] Поведение при offline/ошибке сети: не удалять из UI, пока бэкенд не подтвердил.
